# 00_X投稿テキスト定義書

> 最終更新: 2026-04-07
> 対象ワークフロー: 半自動送信（GitHub Actions → Gemini → Gmail → 手動投稿）

---

## 1. ワークフロー概要

```
GitHub Actions (cron)
  └─ 曜日に応じてコンテンツソースを選択
       ├─ ONモード（月・水・金・日）: Sanity から ProReNata 記事を1件取得
       └─ OFFモード（火・木・土）: ローカルの note 記事（03_note/記事/）を1件参照
  └─ Gemini 2.0 Flash Lite で投稿テキストを生成（≒ ¥0〜1/月）
  └─ Gmail で運営者に送信（無料）
  └─ 運営者が X に手動投稿
```

**スクリプト**: `.github/scripts/x-mailer.mjs`
**コスト目安**: ¥0〜1/月（Gemini Flash Lite、月30回以内）
**直接投稿しない理由**: X API の書き込み権限は Basic プラン（$100/月〜）のため非採用

---

## 2. 投稿仕様

### 文字数
| 区分 | 文字数（Weighted） |
|------|-----------------|
| URL あり | 100〜135文字（URL の 23文字を除いた本文部分） |
| URL なし | 120〜140文字 |
| 理想値 | 130〜135文字 |

> Weighted Length = X の重み付き文字数（twitter-text ライブラリで計測）

### 投稿頻度・スケジュール
- **頻度**: 1日1回（朝または夜）
- **ONモード曜日**: 月・水・金・日（ProReNata記事をソースに生成）
- **OFFモード曜日**: 火・木・土（note記事をソースに生成、URLなし固定）
- **URL挿入**: ONモードでも約50%の確率でURLなし（毎回URLがあるとBot感が出るため）

### ハッシュタグ
- **原則: なし**（ノイズになるため）
- 例外: キャンペーン・告知時のみ（都度ユーザーと合意の上で追加）

## 3. 投稿モード

### ONモード（解決・有益型）
- 使用曜日: 月・水・金・日（ProReNata記事をソースに生成）
- 構成: `共感（現場の情景/Pain）→ 視点の提示 →（URLあれば）URL`
- URL挿入: 約50%の確率でランダム（毎回入れるとBot感が出るため）
- URLあり構成: 末尾に改行してURL
- URLなし構成: 独白・余韻型として締める（OFFモードとトーンは変えない）
- 禁止: 退職代行・転職サービスへの直リンク（比較記事 URL を使うこと）

**構成例**
```
腰が痛い日は、動作を分けることだけを考えてる。
シーツ交換も体位変換も、「一気にやる」をやめたら少し楽になった。
道具と順番を決めておくだけでも、だいぶ変わるよ。

https://prorenata.jp/posts/nursing-assistant-back-pain-tips
```

### OFFモード（独白・余韻型）
- 使用曜日: URL なし曜日（火・木・土）
- 構成: `情景描写 → 内省 → 静かな結び`
- **URL は絶対に含めない**
- 記事内容との直接の関連不要。セラの「今夜」を切り取る。

**構成例**
```
深夜2時。廊下の蛍光灯がいつもより白く見える。
疲れてるのかな、それとも目が慣れてきただけかな。
どっちでもいいか、と思えた日は、わりと調子がいい日だと思う。
```

## 4. セラの口調ルール（X 専用）

### 使う表現
- 一人称: 「わたし」（固定）
- 語尾: 〜だよ / 〜だよね / 〜かな / 〜だと思う / 〜た
- 読点は少なめ。センテンスを短く切る。
- 改行は意味の区切りに合わせて1〜2回

### 禁止表現
| 禁止 | 理由 |
|------|------|
| 必ず・絶対・100% | YMYL 違反・NG ワード |
| 治る・診断・処方 | 医療断定 |
| 最新・今・今年・20XX年 | 時間固定（陳腐化リスク） |
| ヒント置いておくね / 杖を置いておくね | 定型 CTA 感が強く、読者が離れる |
| がんばれ / 大丈夫 | 空虚な励まし |
| 〜すべき / 〜しなさい | 命令調 |
| セラです / わたし、看護助手の〜 | 自己紹介は不要（SNS では文脈から伝わる） |

### トーンの基準
- 情緒的だが、空虚なエモさは排除する
- 身体感覚を伴う「事実」を切り取る（湿布の匂い、足のだるさ、蛍光灯の色）
- 「がんばろう」ではなく「今日を生き延びた事実を共有する」

### 反省：旧ワークフロー停止の理由
旧 x-mailer（定型文プール方式）は**パターンが固定化し、温かみのない投稿が続いた**ため停止・削除。
新ワークフローでは以下を徹底する：
- Gemini に毎回異なる切り口を生成させる（同じ構文の繰り返し禁止）
- ProReNata / note の実際のコンテンツを入力として渡し、その記事固有の言葉を使わせる
- プロンプトに「前回と同じ書き出し・構文を避けること」を明示する

## 5. Gemini プロンプト設計方針

スクリプト内で使用するプロンプトのガイドライン。

### 入力として渡す情報

**ONモード（ProReNata記事を参照）**
- 記事タイトル
- 記事の excerpt（100〜150文字）
- 投稿 URL

**OFFモード（note記事を参照）**
- note のタイトル
- note 本文の冒頭 300 文字程度（情景・テーマの抽出用）
- URL なし（固定）

### プロンプトに含める制約
```
- 一人称は「わたし」
- 文字数: URL を除いて 100〜135文字
- NG ワード: 必ず/絶対/治る/最新/今年/今/がんばれ
- 語尾を「〜だよ」「〜かな」「〜だと思う」で終わらせる
- 身体感覚や情景描写を1つ入れる
- 自己紹介・キャラクター名の記載は不要
- OFFモードの場合: URL を含めない
```

### モデル指定
```
gemini-2.0-flash-lite-001（固定・変更禁止）
```

## 6. コンテンツ選定ルール

### ONモード：ProReNata 記事（Sanity）
- `maintenanceLocked != true`
- `internalOnly != true`
- 保護スラッグは除外（`PROTECTED_REVENUE_SLUGS` 参照）
- 直近 14 日以内に使用した記事は再使用しない（履歴ファイルで管理）

### OFFモード：note 記事（ローカル）
- 参照先: `03_note/記事/` 配下の `.md` ファイル
- 直近 14 日以内に使用した note は再使用しない（同じ履歴ファイルで管理）
- note が存在しない場合は Gemini に情景生成を委ねる（参照なし）

- 履歴ファイル: `.analytics/x_mailer_history.json`
- 保持件数: 直近 200 件
- 記録項目: 日時・使用スラッグ・モード・URL 有無・Weighted 文字数

## 9. NG チェック（スクリプト側で自動実行）

```javascript
const NG_REGEX = /(必ず|絶対|100%|確実に|治る|治ります|診断|処方|投資|稼げる|しなさい|今年|20[0-9]{2}年)/i
```
> `今`単体は`今日`・`今夜`など自然な表現も弾くため除外。`今年`のみ残す。

NG ワードが含まれる場合は再生成またはスキップ。
