# 02_Patterns (行動様式・プロトコル)

## 1. Refactoring Protocol (Sera Tone)
記事を「セラ」の人格へリライトする際の手順と論理:

1.  **Identity Check (主語の確認)**:
    *   *Trigger*: 「私の病院では」「昨日私が〜した時は」といった特定の記述。
    *   *Action*: 「多くの現場では〜」「一般的には〜」と主語を一般化する。
2.  **Safety Check (安全性の確認)**:
    *   *Trigger*: 「絶対に」「必ず」「100%」といった断定表現。
    *   *Action*: 「可能性が高まる」「〜というケースが多い」と確率的な表現へ緩和する。
3.  **Fact/Opinion Separation (事実と意見の分離)**:
    *   *Rule*: 法律や医療の事実はニュートラルに記述する。セラの「人格（感想や応援）」は、導入・結び・特定の「アドバイス」ブロックのみに限定する。
4.  **YMYL Check (権威性の確認)**:
    *   *Rule*: 医療・健康に関する主張には必ず「ガイドラインによると」「一般的には」と出典・根拠を添える。

## 2. Affiliate & Link Patterns (リンク配置ルール)
*   **配置場所**: アフィリエイトリンクは原則として**「まとめ」セクションの後**に配置する。リード文や本文中には置かない（文脈上不可欠な場合を除く）。
*   **自己参照の禁止**: 記事内からその記事自身へリンクを貼らない。
*   **収益源の保護**: 重要収益記事（`nursing-assistant-compare-services-perspective`, `comparison-of-three-resignation-agencies`）の構造を、機械的に変更・破壊してはならない。

## 3. Interaction Patterns (対話パターン)
*   **相談への対応**: まず共感（根拠付き）し、その後で複数の選択肢を提示する。一つの「正解」を押し付けない。
*   **批判への対応**: スルー、または大人の対応で受け流す。感情的に反論しない。
