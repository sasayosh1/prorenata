# Skill Harvesting (Agentic Workflow)

## 概要 (Overview)
**"XP & Skill Tree Updater"**
日誌の「学習」「経験」セクションから、ユーザーが得た「スキル」や「独自ノウハウ（暗黙知）」を収穫し、プロフィール（04_スキルとノウハウ）をアップデートするワークフローです。
AIがユーザーの成長をリアルタイムで認識し、将来の提案に活かせるようにします。

## Trigger
*   **Routine**: `Weekly Review` ワークフローの **Phase 5** として自動実行。

## Prerequisite (事前準備)
*   **対象範囲**: 直近7日間の日誌ファイルすべて（`05_日誌` 配下）。
*   ターゲットファイル: `00_システム\00_UserProfile\04_スキルとノウハウ(Skills).md`
*   **参照元**: `思考と感情のログ` 内の「学び」や「改善点」、および `抽出された情報` セクション。

---

## Agent Action Protocol

### Step 1: Analysis (分析)
**Null Check (重要)**: まずログ全体をスキャンし、新しい「スキル」や「ノウハウ」の記述があるか確認してください。
**情報がない場合**: 「※関連するスキル情報は見つかりませんでした。」と出力し、**ここで処理を終了**してください。

ログ内の `Knowledge Candidates` や `Improvement` セクションを分析し、以下の基準で情報を分類します。

*   **Skill (能力/技術)**: 「できるようになったこと」。客観的に証明可能な技術。（例：Viduでの動画生成、Next.jsでのサイト構築）
*   **Knowhow (ノウハウ/暗黙知)**: 「やってみて分かったコツ」。独自の工夫や哲学。（例：司会時の間の取り方、特定の相手への交渉術）

### Step 2: Categorization (分類)
`04_スキルとノウハウ(Skills).md` の既存カテゴリーと比較し、どこに配置すべきか決定します。
*   既存カテゴリ: `コミュニケーション・表現系`, `クリエイティブ・企画系`, `ビジネス・マーケティング系`
*   該当がない場合は `New Discoveries / Temporary` として扱うか、新設カテゴリを提案する。

### Step 3: Integration (統合)
**重要: 既存の情報を削除・上書きしてはいけません。必ず「追記」を原則とします。**

1.  ターゲットファイルを読み込む。
2.  適切なセクションの末尾に、以下の形式で追記する。
    ```markdown
    *   **[キーワード]** (YYYY-MM追加):
        *   [詳細な説明、得られた知見の要約]
    ```
3.  もし内容が既存の項目と重複・競合する場合（例：以前のノウハウが間違っていた場合）は、古い項目の下に `> [!UPDATE] YYYY-MM-DD: 新たな知見により修正...` といった注釈を加える形で更新する。

### Step 4: Verification (確認)
*   更新内容をユーザーに提示し、「この分類で合っていますか？」と確認を求める場合がある（基本は自動化で良いが、Conflictingな時は聞く）。
