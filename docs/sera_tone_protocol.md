# sera_tone プロトコル：リライト実行規定

本規定は、AIエージェントがProReNataの記事をリライト・校正する際に遵守すべき技術的・倫理的ルールを定義します。

## 1. 原則（The Golden Rules）
1.  **事実改変の禁止**: 給与額、時間、法律用語、制度の仕組み、出典URLなどは一切変更してはならない。
2.  **断定表現の回避**: 「必ず〜になる」「〜しかありません」を、「〜の場合が多い」「〜が一般的です」「選択肢の一つです」に書き換える。
3.  **一人称実体験の削除**: 「わたしの職場では」「現場で見ました」を、「医療現場では一般に」「よくある相談事例では」に一般化する。
4.  **立場の一致**: 白崎セラは「案内役」「公式ナビゲーター」であり、特定の病院の従業員としての発言をしない。

## 2. 構文・構成ルール
- **所感の分離**: 本文（事実）の中にセラの感情や主観を混ぜない。
- **セラの吹き出し/コメントブロック**: 記事の冒頭（期待値調整）または末尾（寄り添い）に限定して配置する。
- **PR表記の自動適用**: 収益リンクが含まれる場合、冒頭に「※当サイトはアフィリエイト広告を利用しています」を挿入する。

## 3. リライト後の「変更サマリー」出力要件
リライト完了後、以下のフォーマットで変更内容を報告すること：

```markdown
### リライトサマリー (sera_tone v1.0)
- **[事実維持]**: 維持された重要データ（給与額、制度名など）
- **[ニュアンス変更]**: 断定を回避した表現の例（例：絶対→一般的）
- **[矛盾修正]**: 削除・修正された一人称体験談
- **[追加要素]**: 挿入された免責事項やPR表記
```

## 4. 禁止ワード・チェック
- 禁止: 「絶対」「必ず」「100%」「わたしが働いている」「運営者のささよし」
- 推奨: 「一般的」「傾向」「〜の可能性がある」「編集部」「案内役のセラ」
