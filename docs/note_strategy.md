# ProReNata Note活用戦略

## 🎯 Noteの役割：「共感とファンのたまり場」
- **Blog (SEO)**: 「検索」で来る人向け。**【解決 7 : 共感 3】**
  - 目的: 新規流入、収益化（アフィリエイト）。
  - 内容: 給料、資格、手順などの「役立つ情報」。
- **Note (Community)**: 「心」で繋がる人向け。**【解決 2 : 共感 8】**
  - 目的: ファンの定着、ブランディング、将来的な有料化（メンバーシップ）。
  - 内容: 現場のリアルな空気感、セラの主観、失敗談、エッセイ。
  - **トーン**: **「親密な友人」モード**。
    - ブログの「丁寧なガイド役（です・ます）」から一歩踏み込み、「〜だよね」「〜だと思う」といった素の言葉遣いを解禁する。
    - 狙い: 「ここだけの姿」を見せることで、読者との心理的距離を縮める（LINEスタンプ等のグッズ展開とも相性が良い）。

## 📝 具体的なコンテンツ案（ブログと差別化）

### 1. 「夜勤明けのひとりごと」（エッセイ）
- **内容**: 現場で感じた「理不尽さ」「小さな感動」「疲れ」をそのまま吐露する。
- **狙い**: 「わかります！」という強力な共感を呼び、読者を「同志」にする。
- **SEO無視**: キーワードを気にせず、文学的・情緒的に書いてOK。

### 2. 「教科書には書けない、グレーな現実」
- **内容**: 建前（ブログ）では書きにくい、現場のドロドロした本音や裏話。
- **狙い**: 「ここでは本音が聞ける」という信頼感。
- **差別化**: ブログは「ホワイト（正しい情報）」、Noteは「グレー（リアルな実態）」。

### 3. ブログ記事の「あとがき / 編集後記」
- **内容**: ブログで解説したテーマについて、「実は私はこう思う」「執筆中にこんなことを思い出した」という個人的な補足。
- **導線**: ブログの文末に「この記事の裏話はNoteで」とリンクを貼る。

## ⚠️ システム連携とコストについて

### 1. NoteのAPI連携（自動投稿）について
**現状、Noteには一般公開されている「記事投稿API」が存在しません。**
そのため、「ボタン一つでNoteに記事が勝手に公開される」という完全自動化はできません。

**推奨ワークフロー:**
1. **下書き自動生成**: Geminiを使って「Note用のエッセイ」をローカル（自分のPC）で生成する。
2. **手動投稿**: 生成されたテキストを、人間がNoteのエディタにコピペして公開する。
   - ※この「ひと手間」が、逆に「誤字チェック」や「写真選び」の品質担保になるため、コミュニティ運営にはプラスです。

### 2. コスト構造（従量課金？）
- **Note側**: 記事投稿は**無料**です。API利用料などもありません。
- **AI側（ProReNataのシステム）**: 記事の**「下書き生成」**にGemini API料金（従量課金）がかかります。
  - エッセイ1本あたりのAIコスト: 約1〜3円程度（激安です）。
