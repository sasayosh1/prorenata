# ProReNata図解定義書 (`00_ProReNata図解定義書.md`)

> **目的**: 公式サイト（prorenata.jp）における複雑な情報の構造化、および読者の意思決定を支援する「実用的な杖」としての図解・チャートを、ブランドの「構造的な美しさ」を守りながら安定して生成するための定義書。

## 1. ビジュアル・リファレンス (Visual Benchmarks)
以下の指針を基本とし、既存の図解（html/svg）は「構造のプロトタイプ」として参照する。
- `11_ブランドビジュアル指針(Brand_Visual_Guidelines).md` (色彩・世界観の基準)
- `01_Values.md` (信頼と誠実さの基準)
- `sample_soft_comparison.svg` (トーン＆マナーの基準)

## 2. デザイン・コアコンセプト (Design Core Concept)
*   **役割**: 「複雑な現状」を「整理された未来」へと変換する案内板。
*   **質感**: 「Nursing Care Style」をベースとした、清潔感と温かみ。
*   **余白**: 情報密度を下げ、読者が「息ができる」レイアウトを徹底する。
*   **アクセシビリティ**: 色覚多様性に配慮し、色だけで情報を区別せず、アイコンやテキストラベルを併用する。

## 3. カラーパレット & タイポグラフィ
*   **Primary**: `Slate 50-900` (基調となる信頼のグレー)
*   **Accent (Positive/Safety)**: `Emerald / Cyan` (安心、推奨、解決)
*   **Accent (Focus/Legal)**: `Indigo / Blue` (法律、専門性、確実性)
*   **Background**: 原則として `Slate 50` または `White`。
*   **Font**: `Noto Sans JP` (ウェイト: Regular 400 / Medium 500 / Bold 700)

## 4. 構成・ブランド表現 (Brand Expression)
*   **一貫性**: サムネイルやUIと連動し、サイト全体で「同じ案内人が作った資料」であることを感じさせる。
*   **要素**:
    *   **角丸**: `Rounded-2xl` (16px) 以上を基本とし、柔らかさを出す。
    *   **ボーダー**: `Slate 200` 前後の極細線。強い囲み線は避ける。
    *   **シャドウ**: `Shadow-xl` など、浮遊感のあるモダンなデザイン。

## 5. 【重要】失敗の定義 (Definition of Failure)
*   **過密な情報**: 1枚の図解に3つ以上の主要トピックを詰め込む（読者が迷う）。
*   **どぎつい配色**: 原色（純粋な赤、青、緑）の使用。ブランドの静謐さを損なう。
*   **非専門的なフォント**: 手書き風や装飾の強すぎるフォントの使用。
*   **構造の欠如**: 視線誘導が設計されていない、要素がバラバラに配置されている。

## 6. 技術仕様 (Technical Specifications)
*   **形式**: SVG（軽量・スケーラブル）または Tailwind CSS を使用した HTML。
*   **レスポンシブ**: モバイルでの視認性を最優先する（文字サイズ、スタック構造）。
